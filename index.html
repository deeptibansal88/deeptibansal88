Key Projects

Conversational AI Driven Automation Workflow

BPMN Softwares can Orchestrates the automation workflow. In the business process automation space, there is a need to integrate BPM platforms with Conversational AI so that end-user interaction and automation can be taken place hand in hand. A separate API is developed in the BPM products to expose the information required by the BPMN workflow to the chatbot. An adapter is written on the chatbot side to incorporate to glue the slots/ action parameters on run time with intent. A separate API is exposed to BPMN products to send end-user notifications to the chatbot.

This way control of the conversation is with BPMN platforms, Whereas conversational AI is performing the interaction and validation of input given by the end-user. This integration has a huge impact on the BPMN vertical of business where interactions with support executives can be eliminated directly by exposing the conversational chatbot to the end-user.

Platforms:Camunda , Infosys Congnitive Automation Stuido , Infosys NIA Chatbot , MS BOT Framework

Microsoft team integrated Chat Opreations

For one of the client, there is a lot of queries and requests like password reset, access requests are coming over an email to L2 support. As a solution we have identified the MS teams as preferred channels and use Azure Bot Services to deploy a multilingual Bot to answer all these frequently asked Queries, PAssword reset requests are taken up by Azure Bot Framework and Automation Anywhere Bot is called by the chatbot using the control tower API to invoke automation. In case the end-user is facing the issue and can not be resolved, the chatbot is integrated with SNOW to raise an incident on behalf of the user.

Multilingualism is achieved by using the language detection API to detect the language of the end-user, which is translated to English by a middleware before sending it to the core Bot. Bot understands the English and responds in English, this response is again translated back to user language by the same middleware. This approach results in significant cost savings

Platforms:MS BOT Framework , Automation Anywhere , Microsoft Team , Microsoft Translator API , Microsoft Language Detection API

Mini Chat (browser based chatbot)

A lot of projects having a small requirement of hosting a chatbot with 30-40 FAQS. Deploying an enterprise chatbot would be overkill and not cost-effective in this scenario. We have come up with an idea of a browser-based UI component. We evaluated various NLP options available with node.js and nave Bayes algorithm for classification. Finally, after evaluating a lot of options we follow below NLP pipeline 1. Tokenization 2. Stop word removal 3. Stemming 4. Term Frequency. Term Frequency is measured against every utterance provided in training data. Finally based on the TF score we provide the configured response of the FAQ.

Technology : Angular 7 , Node.js
